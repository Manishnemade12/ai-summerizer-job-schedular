# SmartCache AI (v2) — Future Ideas

Here are some potential features and improvements we can add to SmartCache AI in future versions (v2):

## 1. Authentication & API Rate Limiting
- Add User Authentication (Google OAuth or JWT)
- Add API Rate Limiting per user (using Valkey/Redis)
- Track which user submitted which job

## 2. Advanced AI Features
- **Batch Processing**: Allow submitting an array of URLs to be summarized together
- **Different Models**: Allow users to choose between Gemini Flash (fast) and Gemini Pro (detailed)
- **Custom Prompts**: Let users customize the tone (e.g., "Summarize this like I'm 5" or "Create a bulleted list")
- **Audio/Video Summarization**: Use Gemini's multimodal capabilities to summarize YouTube videos or audio files

## 3. Database Persistence
- Add PostgreSQL to permanently store job history and summaries (currently Valkey only stores for 24h)
- Create a user dashboard to view past jobs

## 4. Frontend Improvements
- Live updates using **WebSockets** (Server-Sent Events) instead of polling every 1.5s
- Dark/Light mode toggle
- Copy to clipboard & Export to PDF/Markdown buttons

## 5. Technical Backend Enhancements
- Retry mechanisms for failed jobs (Dead Letter Queue)
- Circuit Breaker pattern for Gemini API calls
- Proper logging aggregation (ELK stack or Grafana/Loki)
